    Assignment

    Code Explanation

    As for the html file, there is nothing to show here. All we do is just connect the js file with the code for our fine calculator.

    Now let's move on to the js file. At the very beginning, we simply ask the user to enter 3 variables (current speed, speed limit in the zone and the zone itself where he is driving).

    Next, we check for each of the variables whether the user entered the correct number, and if not, we display a message to him that the parameters he entered are not correct.

    Well, the main code. To begin with, we create a variable for fine and over speed, that will be useful to us further. After that, we just do a lot of checks in order to determine fine and at the end it remains only to print the results.

    Conclusion

    During the development of the fine calculator, the whole class encountered a problem because the site from which we were given the information was updated and there was a lot of data. Because of this we had to do a lot of "if else".

    I wouldn't say the task was difficult, but it was rather long. I had to write a lot of code, because there was no dependency between the numbers. I didn't learn anything new, since I have already encountered everything that was in this task. But it was interesting to work in a team and help other people figure it out.
