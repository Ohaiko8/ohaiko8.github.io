    • Browser Extension

    Assignment

    Develope a chat app written on Node.js.

    Introduction

    When I saw this task in the Intermediate tab, I thought it was some kind of mistake, because I thought that developing browser extensions is much more difficult. But as it turned out, the difficulty is not in porting your application in the browser as an extension, but in writing functionality. But it ended up being a lot easier than I thought. I made a rather banal extension that allows you to download a screenshot of an entire page, or its individual element.

    Code Explanation

    We create a manifest.json file to configure our project. Here we indicate the information that the browser will process in the future when loading our extension. Also indicate that we will use "screen" and "jQuery".

    We create an element "a" on the page and set attributes for it that we will use later.

    We create a function for downloading images. We also call it when our variable changes. How we change it, I will show in the next paragraph.

    As we said, we write that if the user clicks on the button, then the value changes. And then the function for downloading the screenshot is called. We write this code in the panel.js file!

    Next comes the additional functionality that I decided to add for a change. It allows us to take a screenshot of a specific Clement on a page. In order to do this, we need to enter a special mode by pressing the Shift + s keys.

    Next, we write a function to make our selected element highlighted.

    At the end, we take a screenshot of the selected area on double click.

    And css file for our extension at the end.

    Conclusion

    In conclusion, I will say that this task was really very interesting for me, because now I know that making browser extensions is not so difficult. I learned a lot, but I especially want to pay attention to using a manifest file with settings (I have not had to do this before) and working with keystrokes (I did something like this when I worked on Unity, but this was the first time in js).
